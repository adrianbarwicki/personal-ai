# LinkedIn Post — If you still write code in 2026, you do it wrong

**Pillar:** Product building / AI angle
**Audience:** Engineers, CTOs, founders, technical investors
**Goal:** Maximum engagement (controversial), establish AI-native operator credibility
**Format:** Text post — earned authority + clear claim. Carousel optional (see appendix).

---

## Final copy (paste-ready)

> If you still write code by hand in 2026, you're doing it wrong.
>
> I'm saying this as someone who has coded every single day for the last 10+ years. Physics degree. SAP engineer. Years of enterprise software. I love writing code.
>
> And I haven't typed a function from scratch in months.
>
> Here's what changed:
>
> The job stopped being "write the code." The job became "decide what gets built, review what the agent produced, and ship it." The keyboard is the slow part of the loop now.
>
> Engineers who keep typing line by line in 2026 aren't being principled. They're being expensive.
>
> The ones outperforming everyone else aren't the best typers. They're the best at:
>
> – framing the problem precisely enough that an agent can solve it
> – reading diffs faster than they'd write them
> – knowing when the agent is bluffing and when it's right
> – wiring agents into pipelines, not screens
>
> That's a different skill set than what most CVs are still optimised for.
>
> The good news: it's learnable in weeks if you already know how to engineer. The bad news: if you skip it, the gap compounds monthly.
>
> The job hasn't gotten easier. Just unrecognisable.

---

## Why this works

- **Earned authority opener** ("coded every day for 10+ years") — without it, this reads as anti-engineer bait. With it, it's a confession.
- **Specific claim, not vibes** — "I haven't typed a function from scratch in months" is concrete and verifiable.
- **Reframes the controversy** — not "engineers are obsolete" but "the job changed." That's defensible.
- **Polarising but defensible** — engineers who agree will share. Engineers who disagree will comment. Either way: reach.

## Comment-priming follow-up (drop 5 min after post)

> To save the inevitable thread: yes, I still review every line that ships. Yes, I still debug at the keyboard when the agent's stuck. Yes, you absolutely still need to know how the code works — more than ever, actually. The shift isn't "don't understand code." It's "don't be the one producing keystrokes when an agent is faster."
